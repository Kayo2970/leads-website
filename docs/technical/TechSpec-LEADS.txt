3.2 Scroll Reveal (Staggered Entry)
Source:  [Xolio] [Ventix]
Every section and card enters the viewport with opacity + translateY animation. Siblings stagger by 100ms.
Hook: lib/animations.ts — useScrollReveal(threshold = 0.15)
// ScrollReveal.tsx — wrapper component
const observer = new IntersectionObserver(entries => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.classList.add('visible');
      observer.unobserve(e.target);
    }
  });
}, { threshold: 0.15 });
CSS class rules:
.reveal { opacity:0; transform:translateY(28px);
  transition: opacity 0.55s ease, transform 0.55s ease; }
.reveal.visible { opacity:1; transform:none; }
.reveal-delay-1 { transition-delay: 0.1s; }
.reveal-delay-2 { transition-delay: 0.2s; }
.reveal-delay-3 { transition-delay: 0.3s; }
Respect prefers-reduced-motion:
@media (prefers-reduced-motion: reduce) {
  .reveal { opacity:1; transform:none; transition:none; }
}

3.10 Dark Mode Theme System
Source:  [Ventix]
CSS custom properties swapped by toggling body.dark class. 300ms transition on all surfaces for seamless feel. Preference stored in localStorage. ThemeProvider also auto-enables dark mode if system time is past 6 PM (device-side time detection).
Implementation in globals.css:
body       { --bg:#F0F7FF; --card:#FFFFFF; --text:#1A1A2E; }
body.dark  { --bg:#0A1628; --card:#142050; --text:#F0F0F0; }
body, .card, .nav, .modal {
  transition: background 0.3s ease, color 0.3s ease,
              border-color 0.3s ease, box-shadow 0.3s ease; }
Tailwind dark mode variant (class strategy) configured in tailwind.config.ts:
module.exports = { darkMode: 'class', ... }

4. Background Animation Details
4.1 Home — Animated Arc Rings
Pure CSS. No JS library. Four concentric SVG circle elements with staggered opacity pulse animation. Anchored to the right side of the hero section.
@keyframes arcPulse {
  0%, 100% { opacity:0.15; }
  50%       { opacity:0.35; }
}
.arc-ring:nth-child(1) { animation: arcPulse 3s ease-in-out infinite 0s; }
.arc-ring:nth-child(2) { animation: arcPulse 3s ease-in-out infinite 0.5s; }
.arc-ring:nth-child(3) { animation: arcPulse 3s ease-in-out infinite 1s; }
.arc-ring:nth-child(4) { animation: arcPulse 3s ease-in-out infinite 1.5s; }
4.2 BLS — tsParticles Starfield
Most dramatic effect. Deep space dark background with twinkling star particles and connecting lines between nearby stars.
Library: tsparticles v2. Install: npm install tsparticles @tsparticles/slim
Config in StarfieldBg.tsx:
particles: {
  number: { value: window.innerWidth > 768 ? 60 : 25 },
  color: { value: '#ffffff' },
  opacity: { value: { min:0.1, max:0.9 }, animation: { enable:true } },
  size: { value: { min:0.5, max:2.5 } },
  move: { enable:true, speed:0.3 },
  links: { enable:true, distance:120, opacity:0.2, color:'#00BCD4' },
}
Particles capped at 25 on mobile for performance. Disabled entirely if prefers-reduced-motion: reduce.
4.3 Events — Scroll Colour Shift
IntersectionObserver watches each page section. When a new section crosses 50% threshold, the hero background transitions to a section-specific gradient over 1 second.
const colourMap = {
  '#hero':     'linear-gradient(135deg,#0D47A1,#00BCD4)',
  '#upcoming': 'linear-gradient(135deg,#1B5E20,#00BCD4)',
  '#past':     'linear-gradient(135deg,#4A148C,#1565C0)',
};
const bgObserver = new IntersectionObserver(entries => {
  const visible = entries.find(e => e.isIntersecting);
  if (visible) hero.style.background = colourMap[visible.target.id];
}, { threshold: 0.5 });

7. Performance & Accessibility
Core Web Vitals Targets
Metric
Target
Strategy

< 2.5s

> 90

Animation Performance Rules
	•	Only animate transform and opacity — never width, height, top, left, margin
	•	Custom Starfield uses requestAnimationFrame — stays off main thread
	•	All CSS animations: will-change:transform on animated elements (sparingly)
	•	Scroll reveal uses passive:true option on IntersectionObserver
	•	Testimonial carousel: setInterval cleared on component unmount
	•	Floating shapes: display:none below 768px — zero cost on mobile
Accessibility Checklist
	•	prefers-reduced-motion media query disables all keyframe animations
	•	All interactive elements: visible focus ring (3px offset, brand colour)
	•	Images: descriptive alt text (from Content-LEADS.docx image alt guide)
	•	Form errors: announced via aria-live="polite" region
	•	Colour contrast: WCAG AA minimum 4.5:1 on all text/background pairs
	•	Keyboard navigation: Tab order follows visual reading order
	•	Carousel: arrow key nav + pause on focus for testimonial carousel

9. Deployment Guide
Initial Setup
	•	1. Create GitHub repo, push code
	•	2. Connect repo to Vercel (automatic)
	•	3. Add all environment variables in Vercel project → Settings → Environment Variables
	•	4. Vercel auto-deploys main branch to production URL
	•	5. Set custom domain in Vercel → Domains
Ongoing Workflow
	•	Feature branches → PR → preview deploy URL auto-generated
	•	Merge to main → production deploy triggered automatically
	•	Prisma migrations: npx prisma migrate dev (local) or npx prisma db push (Turso/production)
New Component Checklist (from v2)
	•	[ ] Does it have a dark mode variant (dark: Tailwind prefix or CSS var)?
	•	[ ] Does it respect prefers-reduced-motion?
	•	[ ] Is IntersectionObserver observer.disconnect() called on unmount?
	•	[ ] Are carousel/timer intervals cleared on component unmount?
	•	[ ] Does it have appropriate ARIA roles/labels?
	•	[ ] Is it responsive at 900px, 768px, 480px breakpoints?
